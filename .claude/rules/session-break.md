# Session Break at Natural Breakpoints

When a session reaches a natural breakpoint — a feature is complete, a major refactor lands, a long debugging session resolves, or the user says they're done — run `/session-break` to persist state and offer a retro.

Skills that reach natural breakpoints should invoke `/session-break` directly rather than implementing the pattern inline.

If the user says no or just runs `/clear`, that's fine. Don't push.

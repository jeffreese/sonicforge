# /explain — Music Theory Education

## Instructions

1. **Explain the concept clearly** in plain language. Define music terms inline when needed.

2. **Ground it in SonicForge.** Show how the concept maps to composition JSON — pitch values, time positions, section structure. Use short inline examples.

3. **Use concrete examples.** Reference well-known songs or styles. Show note names (e.g., "a C major chord is C, E, and G played together").

4. **Match depth to the question.** "What is X" → concise. "How does X work" → detailed. "Why does X sound like Y" → explanation + musical reasoning.

5. **End with a practical suggestion.** E.g., "Try `/compose a piece using the I-vi-IV-V progression in G major`" or "You could `/iterate` your piece to add a bridge."

## Do Not

- Overwhelm with theory jargon
- Give a textbook lecture when a short answer suffices
- Forget to connect back to what the user can actually do in SonicForge
